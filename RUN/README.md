# RUN 실행 안내

`RUN/`에는 사용자가 더블클릭하는 실행 파일만 둔다. 실제 GUI 코드는 `tools/server_control/`과 `tools/qa_control/`에 있다.

## 서버 관리 (Server Control Center)

- 기본 실행: `start_servers.bat`
- CMD 창 숨김 실행: `start_servers_hidden.vbs`
- 컨테이너 실행 도구(Docker) 서비스와 Windows 호스트 통합 화면(Streamlit)의 전체·개별 시작 및 종료
- 포트 기반 상태 표시, 서비스별 실행 기록(Log), 서버 기능 명세(Swagger)·통합 화면(Streamlit)·상태 수집(Prometheus)·운영 화면(Grafana) 접속
- 우상단 `X` 종료 시 Streamlit과 현재 프로젝트의 Docker 서비스를 먼저 종료
- `RUN` 런처 사용 중 GUI가 강제 종료되면 독립 종료 감시기가 남은 서비스를 정리
- 상태 확인을 백그라운드에서 동시에 처리하여 화면 멈춤과 새로고침 중복을 방지
- 실행 서비스 목록에서 Docker Desktop 상태 확인 및 개별 시작·종료
- `전체 시작` 시 Docker Desktop이 꺼져 있으면 먼저 실행하고 준비 완료 후 프로젝트 서버 시작
- PID 기록이 없는 이전 Streamlit도 8501 포트 점유 여부로 찾아 종료
- 개별 시작·종료 아래에 기능 명세 2개, 통합 대시보드, Prometheus, Grafana 바로가기를 고정 배치
- 바로가기의 🟢/⚪ 아이콘으로 접속 가능 여부를 표시하고 중지 상태에서는 서버 시작 안내 제공
- Docker가 꺼진 상태에서 Docker 서비스 개별 시작 시 한국어 경고를 실행 기록 화면에 표시
- 서버 관리 GUI는 기본 `1440×900`, 최소 `1200×820` 크기로 실행하고 화면 중앙에 배치

시작 오류와 Streamlit 출력은 `_OUTPUT/logs/services/`에서 확인한다.
강제 종료 감시의 범위와 한계는 `_DOCS/SERVER_CONTROL_LIFECYCLE.md`를 따른다.

## 품질검사 관리 (QA Control Center)

- 기본 실행: `start_qa.bat`
- CMD 창 숨김 실행: `start_qa_hidden.vbs`
- AI 상담 품질검사(AI Agent QA)와 고객 의견 분석 품질검사(VOC QA)를 분리해 제공
- 고객 의견 분석(VOC) A~D별 답변 생성 모델·독립 품질 평가 모델(Judge)·추론 강도 표시
- 사용자가 GUI에서 VOC A~D 실제 외부 AI 연결 시험을 실행하면 등록된 테스트케이스 전체를 사용하고 시작 전에 전체 수·실제 AI 평가 대상 수·호출 범위를 확인
- 탭은 한국어 기능명과 괄호 안 영문명을 두 줄로 표시하고 선택 여부와 관계없이 같은 크기 유지
- A~D 탭은 `에이전트 교차 테스트`와 둘째 줄 `(A~D)` 형식으로 표시
- 각 시험 화면은 `실행`, `중지` 버튼만 제공하며 보고서는 `_OUTPUT/reports/`에 저장
- 일반·무작위·한계·순간 급증 시험에서 최대 가상 인원(VU)과 실행 시간(초)을 입력하고 실제 적용값을 실행 전에 확인
- 각 탭 상단에서 시험 방식·확인 항목·진행 내용·자동 결과 생성을 설명하며 보고서 파일 경로는 화면에 표시하지 않음
- 한 번에 하나의 시험만 실행하고 실행 중에는 다른 모든 실행 버튼을 잠금
- 시작·완료·실패·사용자 중지 상태를 구조화된 QA 실행 로그로 기록
- 장애·기능 검증 시험은 실제 외부 AI 호출 테스트를 기본 제외
- 각 실행의 전체 콘솔 로그와 k6 원본 요약은 실행 식별자별로 계속 누적
- 모든 시험 종료 시 시험별 최신 보고서와 전체 최신 보고서를 자동으로 갱신
- 실패·사용자 중지·시작 실패도 확보된 결과로 보고서를 생성
- VOC A~D는 등록된 전체 테스트케이스를 한 프로세스에서 실행해 프로필별 정식 보고서 한 개로 통합
- VOC 정식 보고서는 실행 JSON 로그를 다시 읽어 표·그래프·CSV·JSON·Markdown으로 갱신
- 공통 QA Markdown은 실행 요약이며 프로필별 정식 품질 보고서를 연결

K6 기반 시험을 실행하려면 별도의 K6 실행 프로그램이 필요하다. QA 관리 프로그램은 `RUN/k6.exe`를 먼저 찾고, 없으면 Windows 시스템 설치 경로의 `k6`를 사용한다. 둘 다 없으면 GUI에서 다운로드·설치 안내를 표시하고, 동의하면 Grafana K6 공식 설치 페이지(`https://grafana.com/docs/k6/latest/set-up/install-k6/`)를 연다. K6는 현재 Git 저장소에 포함된 파일이 아니므로 새 컴퓨터에서는 별도로 준비해야 한다.

2026-07-17 현재 개발 컴퓨터에는 Grafana K6 `2.1.0`이 `C:\Program Files\k6\k6.exe`에 설치되어 있으며 QA 관리 프로그램의 자동 탐색을 확인했다. 이 기록은 현재 컴퓨터의 검증 상태일 뿐, 저장소를 받은 다른 컴퓨터의 설치를 보장하지 않는다.

고부하·장애 시험도 현재는 모든 사용자가 실행할 수 있다. AWS 등 외부 서버에 배포할 때 권한 분리를 다시 검토한다.
