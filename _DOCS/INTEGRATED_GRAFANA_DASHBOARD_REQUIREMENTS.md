# 통합 Grafana 대시보드 요구사항

> 작성일: 2026-07-17
> 적용 대상: `_Total` 통합 Streamlit의 운영 상태 화면
> 상태: **Streamlit 하위 탭 4개와 VOC Grafana JSON 2개 구현, 비API 검증 완료**

## 1. 목적

기존 `ai_agent_quality_portfolio` 대시보드처럼 Grafana를 통합 Streamlit 안에서 탭으로 확인한다. 기존 AI 상담용 Grafana 2개를 보존하고 VOC용 Grafana 2개를 추가해 총 4개의 하위 탭으로 구성한다.

2026-07-17에 Streamlit 상위 `모니터링`과 하위 탭 4개를 구현하고 VOC Grafana JSON 2개를 자동 프로비저닝 경로에 추가했다. 기존 Prometheus VOC 지표를 사용하며 Docker 서비스 구성 자체는 변경하지 않았다.

## 2. 확정 화면 구조

```text
모니터링                            ← 통합 대시보드 상위 탭
├─ AI 상담 실시간 운영             ← Grafana 하위 탭 1
├─ K6 성능 부하 시험               ← Grafana 하위 탭 2
├─ VOC 실시간 운영                 ← Grafana 하위 탭 3
└─ VOC QA·A~D 비교                 ← Grafana 하위 탭 4
```

- 기존 포트폴리오의 상위 Grafana 탭을 확장해 `모니터링`을 통합 대시보드의 상위 탭 하나로 둔다.
- 사용자가 상위 `모니터링` 탭을 선택하면 바로 아래에 4개의 Grafana 하위 탭이 표시된다.
- `모니터링 → Grafana 영역 → Grafana 4개`처럼 중간 탭을 하나 더 두는 3단계 구조는 사용하지 않는다.
- Grafana는 단순 외부 링크 한 개로 끝내지 않고 통합 대시보드 내부에 표시한다.
- 각 하위 탭에는 Grafana 원본 화면과 `새 창에서 열기` 버튼을 함께 제공한다.
- Prometheus와 서버 기능 명세 바로가기는 Grafana 하위 탭으로 추가하지 않고 상위 모니터링 화면의 별도 운영 도구 링크로 유지한다.

## 3. Grafana 4개 구성

| 순서 | 하위 탭 | Grafana UID | 구분 | 현재 상태 |
|---:|---|---|---|---|
| 1 | AI 상담 실시간 운영 | `ai-agent-quality` | 기존 보존 | JSON·임베드 코드 존재 |
| 2 | K6 성능 부하 시험 | `k6-performance-test` | 기존 보존 | JSON·임베드 코드 존재 |
| 3 | VOC 실시간 운영 | `voc-live-operations` | 신규 | JSON·임베드 코드 존재 |
| 4 | VOC QA·A~D 비교 | `voc-qa-abcd` | 신규 | JSON·임베드 코드 존재 |

현재 `ops/monitoring/`의 원본과 Grafana 자동 프로비저닝 경로에 같은 내용의 복사본을 두며, 실제 대시보드 종류는 기존 2개와 신규 VOC 2개를 합친 4개다. 파일 개수와 실제 대시보드 종류를 혼동하지 않는다.

## 4. 기존 2개 보존 기준

### 4.1 AI 상담 실시간 운영

- 기존 `AI Agent 발표용 운영 모니터링` 내용을 유지한다.
- 오류율, 요청 수, 응답시간, 채점 결과 등 AI 상담 운영 지표를 표시한다.
- 기본 조회 범위와 밝은 테마 등 기존 발표용 표시 방식을 유지한다.

### 4.2 K6 성능 부하 시험

- 기존 `K6 Performance Test` 내용을 유지한다.
- 가상 사용자 수, 초당 요청 수, 실패 요청, p95 응답시간 등 K6 지표를 표시한다.
- K6 시험 실행 전에는 데이터가 없을 수 있다는 안내를 화면에 표시한다.

## 5. 신규 VOC 2개 기준

### 5.1 VOC 실시간 운영

- 누적 질문 수, 성공률, 현재 처리 중 요청, 평균·p95 응답시간
- 7단계별 평균·p95 처리시간과 에이전트별 성공·실패·시간 초과
- 검색 결과 0건, 동의어 확장, 범위 외 질문 비율
- 실시간 Judge 판정 분포·항목별 평균·채점시간
- VOC API·6개 에이전트·Prometheus 상태와 제공자별 API 성공·실패·재시도

### 5.2 VOC QA·A~D 비교

- 최근 A·B·C·D 총점과 실행 상태
- 프로필별 실행 소요시간과 평가 항목별 평균 점수
- PASS·FAIL·N/A 분포
- A~D 실행 이력과 개선 전후 변화
- 아직 실행하지 않은 프로필은 실패로 표시하지 않고 데이터 없음으로 구분

## 6. 표시 및 예외 처리

- Grafana가 꺼져 있으면 빈 iframe 대신 서버 관리 화면에서 Grafana를 먼저 시작하라는 안내를 표시한다.
- 특정 대시보드가 아직 준비되지 않았으면 `구현 예정` 또는 `데이터 없음` 상태를 명확히 표시한다.
- iframe 로딩과 별개로 새 창 열기 버튼을 항상 제공한다.
- 질문 원문, 고객 내용, 에이전트 출력, 시스템 프롬프트 및 비밀값은 Prometheus 라벨과 Grafana 패널에 넣지 않는다.
- 상세 질문·답변·단계별 원문은 Grafana가 아니라 통합 대시보드의 상세 화면과 보고서에서 확인한다.

### 6.1 페이지 스크롤 통합 구현 결과

> 2026-07-17 실제 화면에서 Streamlit 페이지와 Grafana iframe에 스크롤이 각각 생기는 현상을 확인했다.

- 각 Grafana JSON의 마지막 패널 위치를 기준으로 iframe 전체 높이를 계산한다.
- iframe 자체 스크롤은 끄고 Grafana 패널 전체가 탭 안에 길게 펼쳐지게 한다.
- 사용자는 Grafana 내부 스크롤이 아니라 통합 대시보드의 세로 스크롤 하나로 전체 패널을 확인한다.
- 대시보드별 패널 수와 길이가 다르므로 네 화면에 같은 고정 높이를 강제로 적용하지 않는다.
- Grafana 화면 구조가 바뀌면 JSON 기준 계산 결과도 함께 바뀌어야 한다.

`gridPos.y + gridPos.h`의 최댓값에 행 높이와 여백을 적용해 현재 AI 상담은 3666px, K6는 1082px, VOC 두 화면은 각각 최소 900px로 계산한다. 공식 Streamlit iframe 구성요소의 `scrolling=False`를 적용했으며, 실제 브라우저에서 네 iframe 모두 `scrolling=no`와 AI 상담 마지막 패널까지 로드되는 것을 확인했다.

## 7. 구현 순서

1. 통합 대시보드에 기존 포트폴리오의 Grafana 탭을 확장한 상위 `모니터링` 탭 구성
2. 기존 AI 상담·K6 Grafana 2개를 상위 Grafana 탭 바로 아래의 하위 탭으로 연결
3. VOC 메트릭이 실제 `/metrics`에 노출되는지 확인 및 부족한 항목 보완
4. VOC 실시간 운영 Grafana JSON 작성·프로비저닝
5. VOC QA·A~D 비교용 집계 메트릭과 Grafana JSON 작성·프로비저닝
6. 총 4개 하위 탭의 iframe·새 창 버튼·빈 상태 안내 구현
7. Docker 환경에서 Grafana UID, Prometheus 데이터, 화면 크기와 새로고침 확인

## 7.1 구현 결과

- 기존 UID `ai-agent-quality`, `k6-performance-test`를 유지했다.
- 신규 UID `voc-live-operations`, `voc-qa-abcd`를 추가했다.
- VOC 실시간 운영은 프로필·상태별 요청량, p95 처리시간, Judge 결과와 최근 요청 수를 표시한다.
- VOC QA·A~D 비교는 프로필별 성공·실패 요청, 평균 처리시간과 Judge 성공률을 표시한다.
- Grafana가 중지된 상태에서는 iframe 대신 서버 관리에서 Grafana를 먼저 시작하라는 한국어 안내를 표시한다.
- 브라우저 화면 확인에서는 Grafana 중지 상태 안내와 하위 탭 4개 구성을 확인했다.
- 실제 Docker Grafana 프로비저닝과 실데이터 패널 값 검증은 서버 전체를 시작한 후 별도로 확인한다.
- 실제 Docker Grafana에서 기존 AI 상담 20개 패널과 신규 VOC 탭 연결을 확인했고 iframe 내부 스크롤을 끈 상태로 렌더링했다.

## 8. 완료 기준

- 통합 대시보드에 상위 `모니터링` 탭이 하나 존재한다.
- 상위 `모니터링` 탭을 선택하면 중간 단계 없이 Grafana 하위 탭 4개가 바로 보인다.
- 기존 AI 상담·K6 화면의 기능과 데이터가 유지된다.
- VOC 실시간 운영과 VOC QA·A~D 비교 JSON이 자동 프로비저닝된다.
- 각 탭에서 iframe과 새 창 열기가 모두 동작한다.
- Grafana 중지·데이터 없음·미구현 상태가 빈 화면과 구분된다.
- 실제 지표와 문서의 패널 설명이 일치한다.
