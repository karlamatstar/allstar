# VOC 카드·진행 단계·채팅 화면 후속 개선안

> 작성일: 2026-07-17
> 상태: **구현·비API 회귀·브라우저 반응형 검증 완료**
> 대상: 통합 대시보드의 `VOC 챗봇`과 `VOC 테스트케이스 > 테스트케이스 실행`

## 1. 목적

VOC 챗봇과 테스트케이스 실행의 A~D 카드 상태 표시를 하나의 규칙으로 통일하고, 진행 중 단계 화면의 스크롤 겹침을 제거한다. VOC 채팅 영역은 AI 에이전트 챗봇과 같은 메신저형 고정 높이 화면으로 바꾼다.

이 문서는 후속 구현 기준이자 완료 기록이다. 2026-07-17에 아래 항목을 코드에 반영하고 외부 AI API 호출 없이 구조 테스트와 실제 브라우저 검증을 완료했다.

## 2. 테스트케이스 실행 A~D 상태 표시

### 2.1 상태 배지를 카드 밖으로 분리

데스크톱에서는 네 열 모두 카드 위에 같은 높이의 상태 영역을 둔다.

```text
[실행 중 / 완료 확인 대기]  <- 카드 바깥 상태 영역
┌────────────────────────┐
│ 프로필 제목            │
│ 설명                    │
│ ---------------------- │
│ 생성 모델·평가 모델    │
└────────────────────────┘
[전체 테스트 실행]
```

- 실행 중인 프로필에는 파란색 `실행 중` 배지를 표시한다.
- 완료 상태를 닫기 전에는 주황색 `완료 확인 대기` 배지를 표시한다.
- 선택되지 않은 세 열에도 같은 높이의 빈 상태 영역을 유지해 네 카드의 시작 위치를 맞춘다.
- 카드 내부에서는 상태 배지를 제거해 제목·설명·구분선·모델 정보의 Y축 위치를 네 카드에서 동일하게 유지한다.
- 카드 높이는 상태 배지가 내부에 있을 때보다 줄인다.
- 현재 선택 카드의 파란색 또는 주황색 테두리·배경색은 유지한다.
- 실행 버튼의 위치와 너비는 네 열에서 동일하게 유지한다.
- 모바일처럼 카드가 세로로 쌓이는 화면에서는 선택되지 않은 카드의 빈 상태 영역을 제거해 불필요한 여백을 만들지 않는다.

## 3. 진행 중 7단계 스크롤 안전 여백

현재 진행 중 한 줄 7단계 화면에서는 가로 스크롤 막대가 단계 카드 하단 테두리와 `완료`, `처리 중`, `대기` 문구를 덮을 수 있다.

- 진행 중 `interactive=False` 한 줄 화면과 완료 후 상태 카드·결과 버튼 두 줄 화면의 하단 여백을 분리한다.
- 진행 중 화면에는 단계 카드 하단 상태와 테두리가 완전히 보이는 최소 안전 여백을 둔다.
- 완료 화면은 현재 축소한 결과 버튼·스크롤 막대 간격을 유지한다.
- 가로 스크롤 막대가 내용을 가리지 않으면서도 과도한 빈 공간을 만들지 않아야 한다.

## 4. VOC 챗봇 A~D 선택 상태

VOC 챗봇도 테스트케이스 실행과 같은 선택 상태 표현을 사용한다.

- 외부 API 비용 필수 확인 전에는 A~D 선택 버튼과 채팅 입력을 비활성화한다.
- 필수 확인 전에도 내부 기본 프로필 값은 A로 유지하지만, A 카드의 `선택됨` 배지·파란색 테두리·파란색 배경은 표시하지 않는다. 사용자가 아직 선택을 확정한 것처럼 보이면 안 된다.
- 확인 후 선택되지 않은 프로필 버튼은 테스트케이스 실행 버튼과 같은 강조색의 `선택` 버튼으로 활성화한다.
- 필수 확인란을 체크한 순간부터 내부 기본값 A를 현재 선택 프로필로 시각화한다.
- 현재 선택 프로필의 버튼은 `선택됨`으로 표시하고 비활성화한다.
- 현재 선택 카드 위의 외부 상태 영역에는 `선택됨` 배지를 표시한다.
- 현재 선택 카드에는 테스트케이스 실행의 `실행 중` 카드와 같은 파란색 테두리·배경색을 적용한다.
- 나머지 카드에는 기본 배경과 테두리를 사용한다.
- 데스크톱에서는 네 열의 상태 영역 높이를 같게 하고, 모바일에서는 빈 상태 영역을 제거한다.

### 4.1 상태 표

| 필수 확인 | 카드 | 상태 배지 | 버튼 |
|---|---|---|---|
| 미확인 | 내부 기본값 A, 기본 카드 모양 | 표시하지 않음 | A~D 모두 비활성 |
| 확인 | 선택 프로필 | `선택됨` | `선택됨`, 비활성 |
| 확인 | 비선택 프로필 | 빈 상태 영역 | `선택`, 강조색 활성 |
| 질문 처리 중 | 선택 프로필 | `처리 중`으로 전환 가능 | 전체 선택 버튼 잠금 |

## 5. VOC 메신저형 채팅 영역

VOC 챗봇의 대화 표시를 AI 에이전트 챗봇의 `챗봇과 대화` 화면과 같은 구조로 바꾼다.

- `st.container(height=520, border=True, autoscroll=True)`에 해당하는 고정 높이·테두리·자동 스크롤 구조를 사용한다.
- 사용자 질문과 VOC 답변을 `st.chat_message` 형식의 좌우 대화 흐름으로 표시한다.
- 대화가 길어지면 페이지 전체가 계속 길어지지 않고 채팅 영역 안에서 스크롤한다.
- 새 메시지와 최종 답변이 추가되면 채팅 영역의 최신 메시지가 보이도록 자동 이동한다.
- 대화가 없을 때는 VOC 질문을 입력하면 대화가 표시된다는 빈 상태 안내를 넣는다.
- 프로필, 처리시간, Judge 점수·판정 같은 부가 정보는 해당 답변 말풍선 아래의 보조 문구로 유지한다.
- 7단계 진행 상태와 완료 후 단계 상세는 제거하지 않으며, 채팅 영역과의 위치를 구현 시 실제 화면에서 결정한다.

## 6. 구현 순서

1. 테스트케이스 실행·VOC 챗봇이 함께 사용하는 외부 상태 영역 구성요소 작성
2. 테스트케이스 실행 상태 배지를 카드 밖으로 이동하고 카드 높이 축소
3. 진행 중 한 줄 단계 화면의 스크롤 안전 여백 분리
4. VOC 챗봇 확인 전 선택 버튼 잠금과 확인 후 강조 버튼 적용
5. VOC 선택 카드의 배경·테두리·외부 `선택됨` 상태 표시
6. VOC 대화 목록을 520px 메신저형 내부 스크롤 화면으로 전환
7. 데스크톱·좁은 화면·모바일 브라우저 검증

## 7. 완료 기준

- 테스트케이스 실행 A~D 카드의 제목·구분선·모델 정보·버튼 위치가 모두 일치한다.
- 실행·완료 상태 배지는 카드 내용 위치를 변경하지 않는다.
- 진행 중 단계 카드의 상태 문구와 하단 테두리가 스크롤 막대에 가리지 않는다.
- VOC 챗봇은 확인 전 선택·입력이 잠기고 확인 후 선택 버튼이 강조된다.
- 확인 전에는 기본값 A를 포함한 어떤 카드에도 선택 배지·선택 배경·선택 테두리가 보이지 않는다.
- VOC 선택 카드에는 외부 `선택됨` 배지와 파란색 테두리·배경이 표시된다.
- VOC 대화는 AI 에이전트와 같은 고정 높이 메신저형 영역 안에서 스크롤된다.
- 실제 외부 API를 호출하지 않는 화면·상태 테스트로 먼저 검증한다.

## 8. 2026-07-17 구현·검증 결과

- 테스트케이스 실행과 VOC 챗봇 카드에 `profile-status-slot` 외부 상태 영역을 공통 적용했다. 실행·선택은 파란색, 완료 확인 대기는 주황색 배지를 사용한다.
- 테스트케이스 실행 카드는 데스크톱 17rem, 1200px 이하 19rem로 축소했다. 브라우저에서 네 카드의 상태 영역은 32px, 카드 높이는 272px, 카드 시작 Y축은 모두 같았다.
- 900px 이하에서는 상태가 없는 슬롯을 숨긴다. 600px 검증에서 선택 카드 슬롯은 26px로 유지되고 비선택 카드 슬롯은 0px로 제거됐다.
- 진행 중 단계는 `stage_scroll_progress_*`에 1rem 안전 여백을, 완료 후 선택 버튼 화면은 `stage_scroll_interactive_*`에 0.2rem 여백을 적용해 두 상태의 스크롤 간격을 분리했다.
- VOC 챗봇은 필수 확인 전 내부 기본값 A를 유지하되 네 카드 모두 기본 모양·비활성 `선택` 버튼으로 표시한다.
- 필수 확인 후 A에 외부 `선택됨` 배지, 2px 파란색 테두리, 파란색 배경을 적용하고 A 버튼을 `선택됨`으로 잠갔다. B~D는 활성 `선택` 버튼으로 표시되는 것을 브라우저에서 확인했다.
- VOC 대화 영역을 `height=520`, 테두리, 자동 스크롤이 있는 메신저 컨테이너로 변경했다. 브라우저 계산 높이 520px와 세로 스크롤 `auto`를 확인했다.
- 사용자 질문·응답에 로컬 시각 보조 문구를 추가했고, 완료 응답의 프로필·처리시간·Judge 점수·판정과 7단계 상세는 유지했다.
- 집중 비API 구조·화면 테스트 20건과 전체 회귀 `202 passed, 5 skipped`가 통과했으며 브라우저 콘솔 오류·경고는 없었다. 건너뛴 항목은 현재 VOC gRPC 에이전트 실행 여부 등에 따라 조건부로 제외되는 기존 통합 시험이다. 실제 질문 전송과 외부 AI API 호출은 수행하지 않았다.
