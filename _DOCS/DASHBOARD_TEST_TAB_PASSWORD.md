# 대시보드 테스트 상위 탭 비밀번호 잠금

> 구현일: 2026-07-20
> 적용 화면: `K6 부하 테스트`, `AI 에이전트 테스트케이스`, `VOC 테스트케이스`
> 상태: **구현·자동검사·Docker 브라우저 검증 완료**

## 1. 목적

부하 시험과 실제 외부 AI 호출 가능성이 있는 세 테스트 상위 탭을 일반 운영·보고서 화면과 구분한다. 사용자가 보호된 상위 탭을 선택하면 실제 기능을 렌더링하기 전에 비밀번호를 확인한다.

이 기능은 로컬 시연 환경에서 실수로 시험 화면에 진입하는 것을 줄이는 간단한 화면 잠금이다. 사용자 계정, 권한, 암호화된 비밀번호 저장소, 접속 이력 감사를 제공하는 운영용 인증 체계는 아니다. 외부 공개 또는 AWS 배포 시에는 별도의 인증·권한 체계로 교체해야 한다.

## 2. 보호 범위

다음 세 상위 탭만 비밀번호로 보호한다.

1. `K6 부하 테스트 (K6 Load Test)`
2. `AI 에이전트 테스트케이스 (AI Agent QA)`
3. `VOC 테스트케이스 (VOC QA)`

AI 에이전트 챗봇, VOC 챗봇, 모니터링, 보고서 모음은 기존처럼 바로 표시한다.

## 3. 화면 동작

- 보호된 탭을 처음 선택하면 실제 탭 페이지 대신 `접근 확인` 화면을 표시한다.
- 비밀번호 입력란은 마스킹한다.
- 올바른 비밀번호를 제출하면 같은 브라우저 세션에서 해당 탭을 즉시 연다.
- 비밀번호가 틀리면 `비밀번호가 올바르지 않습니다. 비밀번호를 다시 입력해 주세요.`라고 안내하고 입력란에서 바로 다시 입력할 수 있게 한다.
- K6·AI 테스트케이스·VOC 테스트케이스는 서로 다른 잠금 상태를 사용한다. 한 탭을 열어도 나머지 두 탭은 자동으로 열리지 않는다.
- Streamlit 브라우저 세션이 끝나면 잠금 상태도 사라져 다음 세션에서 다시 확인한다.
- 인증 전에는 보호 대상 렌더링 함수를 호출하지 않아 비활성 탭의 시험 로직도 미리 실행하지 않는다.

## 4. 비밀번호 설정

기본 비밀번호는 사용자 요청에 따라 `1234`다.

```text
DASHBOARD_TEST_TABS_PASSWORD=1234
```

`.env.example`에 기본 설정을 제공하고 Docker Streamlit에는 Compose 환경변수로 전달한다. 기존 `.env`에 항목이 없더라도 Compose 기본값 `1234`를 사용한다. 이후 비밀번호를 바꾸려면 `.env`에 새 값을 지정하고 Streamlit 컨테이너를 다시 생성한다.

```powershell
docker compose up -d --build streamlit
```

비밀번호 원문은 로그·보고서·세션 상태 키 이름에 기록하지 않는다.

## 5. 검증 기준

- 기본값 `1234`만 통과하는지 확인
- 환경변수로 다른 비밀번호를 지정하면 기본값이 거부되는지 확인
- 세 상위 탭이 각각 공통 잠금 렌더러를 사용하는지 확인
- 비밀번호 입력 마스킹과 오답 재입력 문구 확인
- Docker Streamlit에 환경변수가 전달되는지 확인
- 실제 브라우저에서 세 탭의 초기 잠금 화면, 오답 안내, 정답 후 실제 페이지 표시 확인

- K6 탭에서 `1234` 입력 시 잠금 상자가 사라지고 실제 부하 테스트 화면이 표시되는 것 확인
- K6 잠금을 해제한 뒤에도 AI 에이전트·VOC 테스트케이스 탭이 각각 잠긴 상태로 유지되는 것 확인
