# 자동검사 중복 통합 기준

> 작성일: 2026-07-18
> 상태: **1차 통합 완료·전체 비API 회귀 검증 완료**

## 1. 목적

자동검사 개수만 줄이지 않고 동일한 계약을 반복 수집하던 검사를 하나의 진단 단위로 통합한다. 서로 다른 오류 상태, 판정 경계, 실행 수명주기와 제공자별 동작은 독립된 실패 원인을 가지므로 분리 상태를 유지한다.

## 2. 통합 원칙

- 동일한 입력·대상·실패 원인을 검사하는 경우에만 통합한다.
- 반복 대상을 하나의 검사에서 확인할 때는 첫 오류에서 멈추지 않고 누락·오류 목록 전체를 실패 메시지에 표시한다.
- PASS·FAIL·N/A, 503·504·서버 중단, 실행·중지·완료처럼 동작이 다른 상태는 합치지 않는다.
- 통합 전후에 전체 수집 개수와 비API 회귀 결과를 비교한다.
- 과거 문서의 회귀 개수는 당시 실행 이력으로 보존하고 README의 최신 수치만 현재 결과로 갱신한다.

## 3. 2026-07-18 1차 통합

| 대상 | 통합 전 | 통합 후 | 유지한 검증 범위 |
|---|---:|---:|---|
| VOC 6개 에이전트 정적 계약 | 24 | 4 | 파일 존재, Python 문법, 필수 심볼, `__main__` 진입점 |
| MCP 도구 등록 | 5 | 2 | 필수 도구 3개, 보조 도구 2개 |
| 대시보드 의미 디자인 전용 파일 | 4 | 0 | 버튼 색상은 구조 검사, 채팅 입력은 품질 UI 검사, VOC 이동은 진행 검사에 흡수 |
| 한국어 상위 내비게이션 중복 검사 | 1 | 0 | 제목·탭·독립 평가·관리 용어를 상위 구조 검사에 흡수 |
| **전체 수집** | **316** | **288** | 기능별 오류·경계 검사는 유지 |

에이전트와 MCP 목록 검사는 통합 후에도 누락된 파일·심볼·도구를 모두 모아 한 번에 표시한다. 따라서 테스트 수가 줄어도 어느 대상이 실패했는지 확인할 수 있다.

## 4. 검증 결과

집중 검사는 다음 영역을 함께 실행해 `57개 통과`를 확인했다.

- VOC 에이전트 정적 계약
- MCP 등록과 로컬 호출
- 통합 대시보드 구조
- AI·VOC 채팅 품질 UI
- 한국어 용어
- VOC 진행 화면

외부 AI API 호출 파일과 명시적 `end_to_end` 검사를 제외한 전체 회귀 결과는 다음과 같다.

```text
283 passed, 1 skipped, 2 deselected
실행시간: 71.29초
```

건너뜀 1개는 실행 환경 조건이며, 선택 제외 2개는 실제 외부 AI 호출 파일이다. 경고 1개는 Starlette 시험 클라이언트의 기존 사용 중단 예정 안내로 이번 통합 실패가 아니다.

## 5. 실행 명령

전체 수집 개수 확인:

```powershell
.\.venv\Scripts\python.exe -m pytest --collect-only -q
```

외부 AI API를 호출하지 않는 전체 회귀:

```powershell
.\.venv\Scripts\python.exe -m pytest tests `
  --ignore=tests/ai_agent/test_evaluation_pipeline.py `
  --ignore=tests/ai_agent/test_negative_cases.py `
  -k "not end_to_end" -q
```

실제 외부 AI API 검사는 사용자 승인과 지정 범위 확인 후 별도로 실행한다.

## 6. 최종 실행 정책

별도의 빠른 검사·기능별 검사·전체 회귀 단계나 전용 마커 체계는 추가하지 않는다. 이 프로젝트의 작업은 한 번에 마무리하는 흐름이므로 최종 확인에서 위 비API 전체 회귀 명령을 한 번 실행한다.

개발 중 문제 위치를 확인하기 위해 관련 파일만 일시적으로 실행할 수는 있지만, 이를 별도 제품 시험 단계나 고정 실행 체계로 관리하지 않는다. 단순히 표시 개수를 더 줄이기 위해 판정 경계나 오류 상태 검사를 합치지 않으며, 향후 기능 추가로 완전히 같은 검사가 다시 생길 때만 같은 기준으로 통합한다.
