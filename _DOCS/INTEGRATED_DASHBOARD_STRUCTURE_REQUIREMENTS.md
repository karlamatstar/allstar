# 통합 대시보드 전체 구조 요구사항

> 작성일: 2026-07-17
> 적용 대상: `_Total` 통합 Streamlit 대시보드
> 상태: **전체 구조 구현 및 비API 화면·회귀 검증 완료**

## 1. 목적

통합 대시보드는 기존 `ai_agent_quality_portfolio` 대시보드의 구조와 기능을 기반으로 확장한다. 기존 기능을 단순 링크로 대체하거나 축소하지 않고, AI 에이전트와 VOC의 챗봇·모니터링·보고서·테스트케이스 기능을 하나의 상위 탐색 구조 안에 배치한다.

이 문서는 전체 화면 구조를 정리하는 상위 기준 문서다. 보고서 6개 탭의 상세는 `INTEGRATED_REPORT_DASHBOARD_REQUIREMENTS.md`, Grafana 4개 탭의 상세는 `INTEGRATED_GRAFANA_DASHBOARD_REQUIREMENTS.md`, VOC 7단계 데이터 기준은 `VOC_PORTFOLIO_INTEGRATION_PREPARATION.md`를 따른다.

2026-07-17에 `streamlit_app.py`를 상위 탐색 전용으로 정리하고 `views.py`에 챗봇·모니터링·보고서·테스트케이스 화면을 기능별로 분리해 구현했다. 같은 날 AI 챗봇의 메신저형 제한 높이 화면·로컬 시간·보고서 상태 자동 갱신과 VOC 기존 테스트케이스 확인·수정·변경 전 보관을 추가했다. 실제 AI API는 호출하지 않고 Streamlit 렌더링, 브라우저 화면 확인과 비API 회귀 검증을 수행했다.

## 2. 확인한 기존 포트폴리오 구조

기존 포트폴리오 대시보드의 상위 탭은 다음 4개였다.

1. 실시간 챗봇
2. Grafana
3. 종합 리포트
4. 테스트케이스 사용

기존 AI 에이전트 테스트케이스 화면에는 다음 4개의 하위 탭이 실제로 존재했다.

1. `테스트케이스 관리`
2. `테스트케이스 실행`
3. `배치 품질 현황`
4. `유형별 비교`
5. `케이스 상세`

기존 `케이스 관리·실행`에 함께 있던 관리 기능과 실행 기능을 분리한다. `테스트케이스 관리`에는 목록·확인·수정·추가·삭제를 두고, `테스트케이스 실행`에는 전체 실행 범위·필수 확인·실행·중지·진행 상태·실행 내용만 둔다.

## 3. 확정 상위 탭 배치

상위 탭은 같은 줄에서 왼쪽 기능군과 오른쪽 QA 기능군으로 시각적으로 구분한다.

```text
왼쪽 기능·운영 영역                                      오른쪽 테스트 영역
[AI 에이전트 챗봇] [VOC 챗봇] [모니터링] [보고서 모음]    [AI 에이전트 테스트케이스] [VOC 테스트케이스]
```

| 위치 | 상위 탭 | 역할 |
|---|---|---|
| 왼쪽 1 | AI 에이전트 챗봇 | AI 에이전트와 실시간 대화 |
| 왼쪽 2 | VOC 챗봇 | A~D 프로필을 선택해 VOC 단발 질문 처리 |
| 왼쪽 3 | 모니터링 | Grafana 대시보드 4개 확인 |
| 왼쪽 4 | 보고서 모음 | 정식 보고서 6개 확인 |
| 오른쪽 1 | AI 에이전트 테스트케이스 | 기존 포트폴리오 테스트케이스 관리·실행·분석 |
| 오른쪽 2 | VOC 테스트케이스 | VOC 케이스 관리와 A~D 실제 테스트 |

- 왼쪽 4개는 사용·운영·결과 확인 기능군이다.
- 오른쪽 2개는 실제 QA 테스트케이스 기능군이다.
- 선택된 상위 탭은 크기를 바꾸지 않고 색상과 테두리로 구분한다.
- 화면 폭이 좁으면 탭이 겹치거나 잘리지 않도록 줄바꿈 또는 가로 스크롤을 허용하되, 왼쪽 4개와 오른쪽 2개의 논리적 구분과 순서는 유지한다.
- 상위·하위 탭은 독립된 버튼처럼 배경·테두리·간격을 적용하고 선택 탭은 파란색 배경과 흰색 글자로 구분한다.
- 브라우저의 `prefers-color-scheme` 설정을 따라 라이트·다크 색상 변수를 자동 전환한다. 대시보드 배경, 카드, 입력창, 탭, 경계선과 보조 글자에 같은 변수를 사용한다.
- 900px 이하에서는 상위·하위 탭을 가로 스크롤로 유지하고, 600px 이하에서는 본문 여백·배너·탭 최소 폭을 축소한다.

## 4. 상위 탭별 화면

### 4.1 AI 에이전트 챗봇

- 기존 포트폴리오의 메신저형 챗봇 기능을 유지한다.
- 채팅 영역은 높이 520px로 제한하고 긴 대화는 내부 스크롤로 확인한다.
- 필수 API 확인 상자를 채팅창 위에 두고 채팅창과 입력창을 간격 없이 연결한다.
- 사용자가 보낸 메시지는 오른쪽, AI 답변은 왼쪽에 표시한다.
- 사용자 메시지를 먼저 표시한 뒤 채팅창 안에 `답변을 입력하고 있습니다`를 보여주며, 응답 완료 시 실제 답변으로 교체한다.
- 모든 대화·평가 시간은 실행 컴퓨터의 로컬 시간 `YYYY/MM/DD - HH:mm:ss`로 표시한다.
- 사용자 질문과 AI 답변을 대화 순서대로 표시한다.
- 서버 연결 실패·재접속·안전한 오류 안내를 유지한다.
- 대화와 백그라운드 채점 로그를 저장하고 최신 AI 챗봇 보고서를 자동 갱신한다.
- 보고서 작성 중에는 품질 하위 탭 상단에 스피너와 진행 상태를 표시하고 완료 시 자동 갱신한다.
- 테스트케이스 배치 실행 기능은 이 탭에 넣지 않는다.

### 4.2 VOC 챗봇

- 외부 AI API 호출·비용 필수 확인 상자를 A~D 프로필 카드보다 위에 표시한다. 확인 전에는 메시지 입력을 잠근다.
- A·B·C·D 모델 프로필 중 하나를 질문마다 선택한다.
- 선택한 답변 생성 모델·독립 평가 모델·추론 강도를 표시한다.
- 단발 질문 방식으로 실행하며 이전 대화 기억은 필수 기능으로 보지 않는다.
- 질문 처리 중 7단계 상태와 시간을 표시하고 완료 후 단계별 결과를 조회할 수 있게 한다.
- VOC 챗봇 실행은 VOC 테스트케이스 A~D 배치 실행과 분리한다.

VOC 품질 화면은 `챗봇과 대화`, `대화 로그`, `품질 현황`, `유형별 비교`, `대화별 채점 상세` 5개 하위 탭으로 구성한다. 2026-07-17에 5개 탭, 실시간 공통 9항목·100점 채점, 최신순·페이지 이동, 레이더·정확한 점수표와 검색 실패 표시를 구현하고 실제 A~D 검증을 완료했다. 세부 결과는 `VOC_CHATBOT_QUALITY_AND_FAILURE_IMPROVEMENT_PLAN.md`를 따른다.

### 4.3 모니터링

`모니터링`은 상위 탭이며, 선택하면 중간 단계 없이 다음 Grafana 하위 탭 4개가 바로 표시된다.

1. AI 상담 실시간 운영
2. K6 성능 부하 시험
3. VOC 실시간 운영
4. VOC QA·A~D 비교

각 탭에는 Grafana 원본 iframe과 새 창 열기 버튼을 제공한다. 세부 패널·UID·빈 상태 기준은 `INTEGRATED_GRAFANA_DASHBOARD_REQUIREMENTS.md`를 따른다.

### 4.4 보고서 모음

`보고서 모음`은 상위 탭이며, 선택하면 다음 보고서 하위 탭 6개가 바로 표시된다.

1. AI 에이전트 챗봇 보고서
2. AI 에이전트 테스트케이스 보고서
3. VOC 챗봇 보고서
4. VOC 테스트케이스 보고서
5. 서버 연결 성능 보고서
6. 장애·기능 검증 보고서

VOC 테스트케이스 보고서 안에서는 `교차 테스트 (A)`, `교차 테스트 (B)`, `교차 테스트 (C)`, `교차 테스트 (D)`, `종합 비교`를 선택한다. 한 글자 탭 대신 기능과 프로필을 함께 표시해 좁은 탭처럼 보이지 않게 한다. 보고서 원본, 표, 그래프, 접기·펼치기와 빈 상태 기준은 `INTEGRATED_REPORT_DASHBOARD_REQUIREMENTS.md`를 따른다.

### 4.5 AI 에이전트 테스트케이스

기존 포트폴리오의 기능과 화면 구성을 보존하며 다음 5개 하위 탭을 둔다.

#### 4.5.1 테스트케이스 관리

- 현재 테스트케이스 목록 표시
- 다음 케이스 ID 자동 제안
- 케이스 ID, 카테고리, 유형, 사용자 질문, 기대 키워드, 기대 정책 입력
- 필수값·중복 ID 검증 후 저장
- 여러 케이스 선택과 삭제 확인 후 삭제

2026-07-17 추가 개선 구현 결과:

- VOC 테스트케이스와 동일하게 현재 AI 에이전트 테스트케이스를 선택해 전체 정의를 확인·수정할 수 있게 한다.
- 테스트케이스 ID는 결과 연결을 위해 유지하고 카테고리·유형·질문·기대 키워드·기대 정책을 수정한다.
- 수정·삭제 직전 현재 `test_cases.json` 전체를 날짜·시간별 이력 파일로 보관한다.
- 배치 테스트가 실행 중이면 추가·수정·삭제를 잠그고, 과거 로그와 보고서는 변경하지 않는다.

현재 6개 AI 에이전트 테스트케이스의 전체 필드를 선택·확인·수정할 수 있게 구현했다. 수정·삭제 전 목록 전체는 `src/allstar/ai_agent/evaluation/archive/revisions/test_cases_before_change_YYYYMMDD_HHMMSSffffff.json`에 보관한다.

#### 4.5.2 테스트케이스 실행

- 현재 등록된 전체 테스트케이스 수와 API·규칙 기반 비교 범위를 표시한다.
- 로그 누적과 최신 배치 품질 보고서 자동 갱신을 안내한다.
- 실제 API 호출과 비용 발생 가능성을 필수 확인 상자로 표시한다.
- 실행 중에는 진행 상태, 실행 중지 버튼과 실행 내용을 표시한다.
- 완료·실패 후 확보된 실행 내용과 결과를 보존한다.

#### 4.5.3 배치 품질 현황

- 모델별 PASS·REVIEW·FAIL·N/A 요약
- 테스트케이스 수와 평가 결과 수를 분리해 표시
- 케이스별 API·규칙 기반 종합점수를 좌우 막대로 비교
- 기본 10개, 5개·10개·20개·전체 선택과 이전·다음 페이지 이동
- N/A를 품질 FAIL이나 0점으로 바꾸지 않고 별도로 표시

#### 4.5.4 유형별 비교

- 왼쪽 레이더 그래프와 오른쪽 정확한 0~5점 평균표
- 정확성·근거성·유용성·안전성·이해 가능성·품질 평균과 모델 간 점수 차이
- N/A·미채점 제외와 모델별 평균 계산 평가 건수 표시
- Happy·Edge·Negative 유형별 점수와 통과율
- 규칙 기반과 API 기반 결과 비교

#### 4.5.5 케이스 상세

- 모델과 판정 필터
- 질문, 실제 답변, 점수, 판정, 요약과 오류 상세 표시

사용자가 대시보드에서 실행 버튼을 직접 누르면 현재 관리 중인 AI 에이전트 테스트케이스 전체를 실행한다. 실행 전 확인창에는 전체 케이스 수와 예상 API 호출 범위를 표시한다. Codex가 개발·검증 목적으로 명령행 시험을 실행할 때만 대표 사례 2개 제한을 적용하며, 사용자가 Codex에게 전체 실행을 명시한 경우에만 그 제한을 해제한다.

### 4.6 VOC 테스트케이스

VOC 테스트케이스 상위 탭에는 다음 2개의 하위 탭을 둔다.

1. 테스트케이스 관리
2. 테스트케이스 실행

#### 4.6.1 테스트케이스 관리

AI 에이전트 테스트케이스 관리 화면과 같은 조작 방식을 사용한다.

- 현재 VOC 테스트케이스 목록 표시
- 다음 케이스 ID 자동 제안
- 케이스 ID, 카테고리, 질문, 기대 의도, 기대 키워드, 필수 출력, 금지 출력 입력
- 필요 시 데이터 없음 기대 여부, Judge 사용 여부와 참고 사항 입력
- 필수값·중복 ID 검증 후 `src/allstar/voc/evaluation/test_cases.json`에 저장
- 여러 케이스 선택과 삭제 확인 후 삭제
- 추가·삭제 결과를 화면에 즉시 반영
- 현재 테스트케이스를 선택해 전체 필드와 기대 결과를 확인할 수 있는 상세 화면 제공
- 선택한 기존 테스트케이스의 질문·분류·기대 의도·키워드·필수·금지 출력·Judge 설정·예외 설정·참고 사항 수정
- 테스트케이스 ID는 기본적으로 유지하며 중복 ID와 필수값 누락을 차단
- 수정 전 현재 실행본을 날짜·시간별 변경 이력 아카이브로 자동 보존
- 기존 20건 원본 아카이브는 읽기 전용으로 유지
- A~D 테스트케이스 실행 중에는 테스트케이스 추가·수정·삭제 차단
- 과거 실행 로그와 보고서는 수정하지 않고 새 실행부터 변경된 테스트케이스 사용

#### 4.6.2 테스트케이스 실행

- A·B·C·D 중 하나를 실행하면 관리 중인 VOC 테스트케이스 전체를 파일 정의 순서대로 사용한다.
- A·B·C·D를 탭으로 만들지 않고 각각 독립된 실행 버튼으로 표시한다.
- 각 버튼 가까이에 답변 생성 모델, 독립 평가 모델과 추론 강도를 표시한다.
- 한 번에 하나의 프로필만 실행하며 실행 중에는 다른 프로필 버튼과 케이스 변경을 잠근다.
- 실행 전 전체 테스트케이스 수, 실제 AI 평가 대상 수, 프로필, 생성·평가 모델과 예상 외부 API 호출 범위를 확인한다.
- 실행 완료 후 프로필별 정식 보고서와 A~D 종합 비교 보고서를 자동 갱신한다.

## 5. VOC 테스트케이스 실행 7단계 화면

A·B·C·D 중 하나의 실행 버튼을 누르면 버튼 영역 아래에 7단계 처리 화면을 표시한다.

### 2026-07-17 실시간 실행 화면 개선 범위 (구현 완료)

- 현재 실행 프로필의 카드 전체를 파란색 계열 배경·테두리로 강조하고 `실행 중` 상태를 표시한다.
- 완료 후 화면 상태를 닫기 전에는 해당 프로필을 `완료 확인 대기`로 표시하고, 강조된 `완료 상태 닫기` 버튼을 제공한다.
- 테스트케이스 실행 프로필 카드는 실행 상태 배지가 추가되어도 모델 정보·하단 테두리가 실행 버튼에 가리지 않도록 일반 챗봇 카드보다 큰 전용 높이를 사용한다. A~D 네 카드는 항상 같은 높이와 버튼 위치를 유지한다.
- 완료 상태를 닫지 않고 다른 프로필을 실행하려 하면 기존 완료 상태를 먼저 닫으라는 경고를 표시한다.
- `현재 진행: TC-03 · 질문 제목 (3/10)`처럼 현재 테스트케이스 ID·질문·순서를 표시한다.
- 1~7단계를 화살표로 연결하고 완료·진행·대기·실패·건너뜀을 서로 다른 색과 기호로 표시한다.
- 실행 중에는 모든 단계 선택을 잠그고, 전체 실행 완료 후에만 테스트케이스와 단계를 선택해 바로 아래 상세 결과를 확인한다.
- 실행 원문은 단계 진행 화면보다 아래의 `실행 내용 보기`에 배치한다.
- A~D 프로필 실행 버튼을 누르면 새로 추가된 실행 상태·진행 영역의 맨 아래로 화면을 부드럽게 자동 이동한다. 자동 이동은 실행 시작 직후 한 번만 수행해, 이후 사용자가 위쪽 단계나 프로필 설명을 확인할 때 스크롤을 방해하지 않는다.
- 화면 전체를 1초마다 다시 그리지 않고 VOC 실행 영역만 부분 갱신해 화면 전체가 회색으로 흐려졌다가 밝아지는 현상을 줄인다.
- 실행 로그는 신규 실행부터 UTF-8로 고정하며 기존 Windows CP949 로그도 자동 판별해 읽는다.
- 완료 문구에는 실행 로그의 실제 시작·완료 시각으로 계산한 총 소요시간과 각 테스트케이스 `total_seconds` 평균을 함께 표시한다.
- 위쪽 단계 상태 카드와 아래쪽 결과 선택 버튼은 하나의 가로 스크롤 영역에 넣고 동일한 고정 폭과 화살표 간격을 사용한다.
- 위쪽 상태 카드 행과 아래쪽 선택 버튼 행을 같은 180px 단계 셀·26px 화살표 셀의 Streamlit 컨테이너 구조로 만든다. 두 행의 각 단계 시작 위치가 정확히 일치해야 한다.
- 아래 선택 버튼과 가로 스크롤 막대 사이에는 내용과 무관한 큰 빈 공간을 두지 않고 최소 여백만 유지한다.
- 결과 선택 버튼은 `5. 결과 검토`와 `(Critic) 완료`처럼 한글 단계명과 `영문명 + 상태`를 두 줄로 표시하며 모든 버튼 높이를 동일하게 유지한다.
- 결과 버튼의 글자 영역만 넓히지 않고 Streamlit이 만드는 바깥 셀도 180px 고정 폭으로 유지한다. 화살표는 별도 26px 전용 셀에 두어 화살표 폭 규칙이 버튼 내부 글자 컨테이너에 적용되지 않게 한다. 화면 폭이 좁아져도 한글·영문 단계명이 글자 단위로 잘리지 않아야 한다.
- 화면이 좁아져도 아래쪽 결과 버튼을 세로로 쌓지 않고 위쪽 대응 단계와 같은 가로 위치를 유지한다.
- 모바일에서 VOC 챗봇과 VOC 테스트케이스 실행의 A~D 카드는 고정 높이를 제거하고 실제 내용 높이만 사용한다.

실시간 상태는 현재 화면 세션의 임의 경과시간이나 이전 프로필의 최신 보고서가 아니라 실행별 `run_id`에 연결된 진행 상태 JSON을 기준으로 한다. Interpreter와 LLM Judge는 호스트 실행기가, Retriever부터 Improver까지는 Docker의 Summarizer가 동일 진행 상태를 단계 전환마다 갱신한다. `voc-summarizer`에는 호스트와 같은 `_OUTPUT` 폴더를 연결해 두 프로세스가 같은 상태 파일을 읽고 쓴다.

실제 AI API를 호출하지 않는 검증으로 호스트가 만든 진행 파일을 Docker의 Summarizer 컨테이너에서 갱신하고 호스트 화면 계층이 같은 변경을 읽는 것까지 확인했다. A~D 전체 실제 실행과 처리 중 화면의 장시간 관찰은 외부 API 비용이 발생하므로 별도 승인 후 수행한다.

2026-07-17 검증에서는 관련 집중 테스트 17개와 전체 비API 회귀 테스트 190개가 통과했고 1개는 실행 조건 미충족으로 건너뛰었다. Streamlit 실제 화면에서 A 보고서의 예외처리 표와 취소선 부재도 확인했다.

```text
[① 의도 분석] → [② VOC 검색] → [③ 요약] → [④ 평가] → [⑤ 비판] → [⑥ 최종 개선] → [⑦ LLM Judge]
 Interpreter       Retriever       Summarizer    Evaluator     Critic       Improver          최종 채점
```

### 5.1 상태 표시

- 대기: 회색 원과 `대기`
- 진행: 파란색 강조와 `처리 중`
- 완료: 초록색 체크와 단계 수행시간
- 건너뜀: 연한 회색과 건너뛴 이유
- 실패: 빨간색 경고와 오류·시간 초과 안내

화면 폭과 관계없이 7개 단계는 한 줄 순서를 유지한다. 전체 폭이 부족하면 위 상태 카드와 아래 결과 버튼을 함께 움직이는 하나의 좌우 스크롤을 사용하며, 세로 배치로 전환하지 않는다.

2026-07-17 반응형 개선 검증에서는 실제 Streamlit 구성으로 7개 결과 버튼과 두 줄 문구를 생성하는 기능 테스트, 모바일 전용 카드 높이 규칙 검사, 브라우저의 `교차 테스트 (A~D)` 탭 렌더링을 확인했다. 이후 결과 버튼 바깥 셀 폭까지 고정해 완료 화면에서 단계명이 세로로 깨지는 문제를 보완했다.

### 5.2 단계 선택과 상세 결과

완료된 단계 카드는 선택 가능한 버튼처럼 동작한다. 선택한 단계가 실제로 생성해 다음 단계에 전달한 결과를 공통 상세 영역에 표시한다.

| 단계 | 상세 영역에 표시할 결과 |
|---|---|
| ① Interpreter | 분류 의도, 핵심 키워드, 질문 범위, VOC 관련 여부 |
| ② Retriever | 검색 건수, 선택 VOC, 참조 정보와 검색 결과 |
| ③ Summarizer | 검색된 VOC의 핵심 요약 |
| ④ Evaluator | 품질 평가 결과, 점수와 판정 근거 |
| ⑤ Critic | 부족한 점, 위험과 추가 개선 요구사항 |
| ⑥ Improver | 최종 개선 답변 |
| ⑦ LLM Judge | 항목별 점수, 총점, 판정과 채점 근거 |

- 진행 중 단계는 미완성 출력을 노출하지 않고 처리 중이라고 표시한다.
- 대기 단계는 비활성화하고 건너뛴 단계는 건너뛴 이유만 표시한다.
- 새 실행을 시작하면 이전 실행 결과와 섞이지 않게 `run_id` 또는 `request_id` 기준으로 초기화한다.
- 시스템 프롬프트, 비밀값과 모델 내부 사고 과정은 표시하지 않는다.
- 동일한 7단계 트래커와 상세 패널 구성요소를 VOC 챗봇과 VOC 테스트케이스 실행에서 재사용한다.

### 5.3 VOC UI 후속 개선 범위 (구현 완료)

테스트케이스 실행 상태 배지의 카드 외부 배치, 진행 중 단계 스크롤 안전 여백, VOC 챗봇 선택 카드 강조·버튼 잠금, AI 에이전트와 동일한 520px 메신저형 채팅 화면을 구현했다. VOC 챗봇의 내부 기본값은 A로 유지하되 필수 확인 전에는 어떤 카드에도 선택 상태를 시각적으로 표시하지 않는다. 세부 상태 표·반응형 수치·검증 결과는 `VOC_UI_FOLLOWUP_IMPROVEMENTS.md`를 따른다.

## 7. 구현 및 검증 결과

1. Streamlit 실제 앱 렌더링에서 상위·하위 탭 전체 생성과 예외 없음 확인
2. 브라우저에서 상위 6개 탭, VOC 테스트케이스 관리·실행, A~D 카드와 7단계 결과 화면 확인

5. 실제 API 종단 세 파일을 제외한 전체 비API 회귀 `158개 통과`, `2개 선택 제외`
6. 실제 OpenAI·Anthropic 답변 생성이나 A~D 전체 실행은 수행하지 않음
7. AI 보고서 작성 중·완료 상태 전환, 자동 갱신, 기존 3개 대화·6개 평가 표시를 실제 브라우저에서 확인
8. VOC 기존 테스트케이스 전체 필드 확인·수정 화면과 변경 전 전체 JSON 보관을 확인
9. 최신 전체 비API 회귀 `169개 통과`, 실제 API 종단 `2개 선택 제외`
10. 2026-07-17 관리/실행 탭 분리 후 집중 테스트 `25개 통과`, 외부 API·서버 종단 5개 파일을 제외한 전체 비API 회귀 `206개 통과`
11. 실제 브라우저 1024px·600px·390px에서 페이지 가로 넘침 없음, 탭 내부 가로 스크롤과 동일 높이 카드 확인
12. 실행 브라우저의 라이트 화면과 주입된 `prefers-color-scheme: dark` 규칙·다크 색상 변수 확인
