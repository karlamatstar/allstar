# AI 에이전트 챗봇 장애 시험 버튼 기준

> 작성일: 2026-07-18
> 대상: 통합 대시보드 `AI 에이전트 챗봇 > 챗봇과 대화`
> 상태: **구현·실제 서버 수명주기 검증 완료 (2026-07-18)**

## 1. 목적

일반 질문 문구에 숨은 특수어를 넣어 장애를 발생시키지 않고, 채팅 입력창 아래의 명시적인 시험 버튼으로만 503·504·실제 채팅 서버 중단을 재현한다. 시험 질문과 오류 응답은 실제 챗봇 대화 로그·품질 보고서에 포함하되 AI 답변 품질 실패로 계산하지 않는다.

## 2. 버튼 구성

AI 에이전트 채팅 입력창 아래에 같은 폭의 버튼 3개를 둔다.

1. `503 서비스 이용 불가 시험`
   - 현재 등록된 AI 테스트케이스 중 하나를 무작위 선택한다.
   - 선택 질문을 사용자 메시지처럼 채팅창에 표시한다.
   - 전용 장애 시험 API가 OpenAI를 호출하지 않고 즉시 HTTP 503을 반환한다.
2. `504 시간 초과 시험`
   - 현재 등록된 AI 테스트케이스 중 하나를 무작위 선택한다.
   - 선택 질문을 사용자 메시지처럼 표시한다.
   - 전용 장애 시험 API가 10초 대기 후 HTTP 504를 반환한다.
3. `채팅 서버 중단 시험`
   - 현재 등록된 AI 테스트케이스 중 하나를 무작위 선택한다.
   - 버튼을 한 번 누르면 선택 질문을 사용자 메시지처럼 즉시 표시하고 현재 프로젝트의 `portfolio-api` 컨테이너만 종료한다.
   - 종료 후 실제 채팅 요청의 연결 실패를 확인하고 오류를 기록한다.

시험 버튼은 사용자가 직접 입력한 일반 질문과 분리한다. `퇴근`, `강사` 등 특정 단어만으로는 장애가 발생하지 않는다.

## 3. 로그·채점·보고서

세 시험은 `_OUTPUT/logs/ai_agent/live/`의 실제 챗봇 로그에 저장한다.

- 대화 로그
  - `request_id`
  - 무작위 선택 테스트케이스 ID와 질문
  - 오류 안내 답변
  - `status=error`
  - 장애 유형 `http_503`, `http_504`, `server_down`
  - HTTP 상태 또는 연결 실패 상태
  - 오류 응답까지 걸린 시간
- 독립 품질평가 로그
  - API 기반·규칙 기반 2행을 모두 기록
  - `overall_decision=N/A`
  - 총점과 5개 품질 항목 점수는 `N/A`
  - 평가 의견에는 인프라·통신 장애로 채점할 수 없음을 기록
- 최신 AI 에이전트 챗봇 보고서
  - 장애 대화를 실제 대화 수와 채팅 목록에 포함
  - `N/A·확인 필요 결과`에 장애 유형·HTTP 상태·오류 내용을 표시
  - PASS·REVIEW·FAIL 통과율과 평균점수 계산에서는 제외

정상 품질 실패와 장애로 인한 평가 불가를 혼합하지 않는다. 장애 시험은 OpenAI Judge를 호출하지 않는다.

### Grafana 운영 지표

- 강제 503·504·채팅 서버 중단도 실제 서비스 에이전트 장애와 같은 운영 카운터에 기록한다.
- 별도 `fault_test` 계열로 분리하지 않고 `service_agent`의 `retry`와 `unavailable`에 합산한다.
- 현재 실제 서비스 에이전트 정책과 동일하게 장애 1건당 `retry 3회`, 최종 `unavailable 1회`를 기록한다.
- 세 장애는 `chat_requests_total{status="error"}`에도 포함해 `오류율 (%) (503·504·서버다운)` 패널에 표시한다.
- 운영 오류 카운트와 별개로 품질 로그·보고서 판정은 계속 `N/A`이며 PASS·REVIEW·FAIL 집계에서 제외한다.
- 서버 중단 장애는 API가 꺼진 상태에서 파일 로그에 먼저 저장하고, 재접속으로 서버가 시작될 때 누적 로그에서 운영 카운터를 복원한다.

## 4. 서버 중단·재접속

- 중단 대상은 Compose 서비스 `portfolio-api` 하나로 제한한다.
- VOC, Prometheus, Grafana, Streamlit과 Docker Desktop은 종료하지 않는다.
- 서버 중단 상태에서는 일반 채팅 입력과 세 장애 시험 버튼을 잠근다.
- 서버 중단 결과를 AI 답변 말풍선으로 표시하지 않는다.
- 채팅창 중앙의 시스템 경고 상자에 `채팅 서버 중단` 제목과 답변 불가 안내를 표시하고, 바로 아래에 `채팅 서버 재접속` 버튼을 둔다.
- Docker Streamlit은 내부 전용 `service-control` 브리지에 `portfolio-api` 시작을 요청하고 `/health`가 HTTP 200이 될 때까지 제한시간 내 확인한다.
- 시작 명령 성공만으로 복구 완료로 판단하지 않고 실제 Health 응답을 확인한다.
- 서버 중단·재접속 시작·성공·실패 이벤트는 별도 장애 이벤트 JSONL에도 누적한다.

Streamlit에는 Docker 소켓을 연결하지 않는다. Docker 소켓은 호스트 포트를 공개하지 않는 내부 `service-control` 컨테이너에만 연결하고, 이 브리지는 현재 Compose 프로젝트의 `portfolio-api`와 `voc-api` 시작·중단만 허용한다. 세부 보안 경계와 AWS 전환 기준은 `CHAT_SERVICE_CONTROL_BRIDGE.md`를 따른다.

## 5. 화면 상태

- 세 버튼 바로 위에 `외부 AI API를 호출하지 않으므로 API 비용이 발생하지 않습니다` 안내를 표시한다. 이 안내는 일반 채팅용 API 비용 확인 체크박스와 장애 시험이 서로 독립임을 분명히 한다.
- 장애 요청 처리 중에는 일반 입력과 모든 장애 버튼을 잠근다.
- 504 시험 중에는 채팅창 안에서 `답변을 입력하고 있습니다` 상태를 10초 동안 표시한다.
- 503·504 결과는 AI 답변 위치에 오류 코드와 평가 `N/A` 안내를 표시한다.
- 서버 중단 처리 중에는 답변 말풍선 대신 중앙 시스템 상태 상자를 표시한다.
- 서버 중단 결과는 중앙 경고 상자와 재접속 버튼으로 유지한다.
- 재접속 성공 후 중앙 성공 상자를 표시하고 입력과 장애 버튼을 다시 활성화한다.

## 6. 검증 기준

- 임의 질문이 실제 `test_cases.json` 현재 목록에서 선택된다.
- 503은 즉시, 504는 10초 이후 해당 HTTP 상태를 반환한다.
- 두 오류가 실제 대화·채점 JSONL과 최신 Markdown·CSV 보고서에 반영된다.
- 장애 결과가 FAIL 수·통과율·평균점수에 포함되지 않는다.
- 서버 중단 버튼이 `portfolio-api`만 실제 종료한다.
- 버튼 한 번으로 무작위 질문 표시와 서버 중단을 시작한다.
- 중단 중 입력 잠금·중앙 경고·재접속 버튼이 보인다.
- 재접속 후 `/health` 200과 일반 채팅 입력 복구를 확인한다.
- 특정 일반 질문 단어만으로 장애가 발생하지 않는다.

## 7. 검증 결과

- 실제 503 응답과 일반 대화 1건·N/A 채점 2건·최신 보고서 갱신을 확인했다.
- 실제 504 응답이 약 10초 뒤 반환되고 동일한 저장 흐름을 사용하는 것을 확인했다.
- `portfolio-api` 단독 중단 후 AI Health 연결 실패를 확인했으며 VOC·Prometheus·Grafana는 계속 실행됐다.
- 재접속이 컨테이너를 다시 시작하고 `/health` HTTP 200을 확인했다.
- 실제 대시보드에서 버튼을 한 번 누르면 무작위 테스트케이스 질문이 사용자 말풍선으로 먼저 표시되고, 답변 말풍선 없이 중앙 중단 경고와 재접속 버튼이 나타나는 것을 확인했다.
- 중단 중에는 장애 시험 버튼 3개가 모두 잠기며, 재접속 완료 후 중앙 성공 안내와 함께 다시 활성화되는 것을 확인했다.
- 누적 강제 장애 11건을 서버 재시작 후 실제 `service_agent retry 33회`, `unavailable 11회`, 오류 요청 11건으로 복원하고 Grafana에 같은 운영 장애로 노출되는 것을 확인했다.
- Docker Streamlit 화면에서 실제 중단·잠금·재접속 완료 자동 전환을 확인했다.
- 브리지 집중 자동검사 17개와 외부 AI 실행 경로를 제외한 전체 회귀 294개가 통과했다. 1개는 환경 조건으로 건너뛰고 2개는 선택에서 제외됐으며 실제 외부 AI 모델 호출은 없었다.
