# AI 에이전트 지식 원본 자동 동기화

> 작성일: 2026-07-17
> 상태: **구현·비API 회귀·실제 API 검증 완료**

## 1. 발견한 문제

AI 에이전트의 규칙 기반 답변과 API 기반 답변은 같은 `COURSE_KNOWLEDGE`를 가져오지만 실제 사용 범위가 달랐다. 규칙 기반은 `drop_out_policy`를 직접 조회한 반면, API 답변 모델의 시스템 안내는 과정명·시간·출결·수료·프로젝트·취업지원만 수동으로 나열했다.

그 결과 `중도 포기 가능한가요` 질문에 규칙 기반은 `수강 시작 후 하차 가능, 재합류 불가`를 답했지만 API 기반은 정보가 없다고 답했다. 2026-07-17 저장 로그에서 같은 질문이 두 번 모두 API FAIL, 규칙 기반 PASS로 반복되어 모델의 우연한 편차가 아닌 지식 전달 결함임을 확인했다.

## 2. 개선 내용

- `COURSE_KNOWLEDGE` 전체를 안내문으로 변환하는 `format_course_knowledge()`를 추가했다.
- API 답변 모델과 독립 채점 모델이 같은 변환 결과를 사용한다.
- 지식 항목을 추가하면 `service_agent.py`의 시스템 안내를 별도로 수정하지 않아도 자동 포함된다.
- 사용자용 한글 항목명이 없더라도 내부 키로 포함해 지식 누락을 방지한다.
- 모든 지식 값이 API 안내에 포함되고 답변·채점 모델의 지식 문자열이 같은지 비API 테스트로 검증한다.

## 3. 규칙 기반 답변과의 차이

규칙 기반 답변은 질문 키워드를 특정 지식 항목에 연결하는 비교용 대조군이다. 새로운 질문 유형을 규칙 기반에서도 답하게 하려면 해당 키워드 연결은 추가해야 한다. 다만 API 시스템 안내는 지식 원본 전체를 자동 반영하므로 더 이상 별도로 수동 복사하지 않는다.

## 4. 운영 반영

지식은 Python 모듈을 불러올 때 시스템 안내로 만들어진다. 로컬 실행 또는 Docker 실행 중 지식을 바꾼 경우 AI 에이전트 API 프로세스를 재시작해야 새 지식이 반영된다. Docker 이미지에 소스가 복사되는 배포 방식이라면 이미지 재빌드 후 재시작한다.
