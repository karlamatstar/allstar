# 로그 날짜 분리와 자동 압축 기준

> 작성일: 2026-07-18
> 적용 대상: AI·VOC 라이브 로그와 AI·VOC 테스트케이스 원본 로그
> 상태: **구현·기존 로그 마이그레이션·비API 회귀·Docker 연동 검증 완료**

## 목적

원본 증적은 잃지 않으면서 장기간 누적되는 텍스트 로그의 저장 공간을 줄인다. 보고서와 Grafana 조회를 훼손하지 않도록 원본 JSONL과 GZIP 로그를 같은 데이터처럼 읽는다.

## 보존 정책

- `최근 5일`이 아니라 로그가 실제 존재하는 **최근 활동 날짜 5개**를 원본으로 유지한다.
- 로그가 없었던 날짜는 5개 계산에서 제외한다.
- 현재 기록 중인 오늘 로그는 항상 원본으로 유지한다.
- 최근 활동 날짜 5개보다 오래된 종료 날짜 로그만 `.jsonl.gz`로 압축한다.
- 활동이 오랫동안 없어도 마지막 활동 날짜 5개의 원본은 계속 유지한다.
- AI와 VOC의 최근 활동 날짜는 서로 독립적으로 계산한다.
- 테스트케이스는 AI 실행 계열과 VOC 프로필 A·B·C·D별 최근 완료 실행 5개를 원본으로 유지한다.
- 완료 여부를 증명하는 manifest가 없는 과거 VOC 실행 폴더는 자동 압축하지 않는다. 새 실행과 완료 manifest가 있는 기존 실행부터 안전하게 관리한다.

## 저장 구조

```text
_OUTPUT/logs/ai_agent/live/
├─ conversations/YYYY-MM-DD.jsonl[.gz]
├─ judgments/YYYY-MM-DD.jsonl[.gz]
└─ faults/YYYY-MM-DD.jsonl[.gz]

_OUTPUT/logs/voc/live/
├─ conversations/YYYY-MM-DD.jsonl[.gz]
└─ judgments/YYYY-MM-DD.jsonl[.gz]
```

AI 대화·채점·장애 로그는 한국 로컬 날짜 기준으로 분리한다. 질문과 백그라운드 채점이 자정을 사이에 두고 서로 다른 파일에 기록되더라도 `request_id`로 연결한다.

## 압축 제외 대상

다음 자료는 화면과 자동 갱신에서 즉시 읽어야 하므로 압축하지 않는다.

- 최신 정식 Markdown·CSV·JSON 보고서
- 최신·이력 PNG 그래프
- 보고서와 실행 manifest
- 현재 실행 진행 JSON과 터미널 로그
- 오늘 기록 중인 라이브 로그
- 계열별 최근 5개 테스트케이스 실행 원본 로그

## 안전한 압축 절차

1. 기록이 끝난 날짜 또는 완료 manifest가 있는 테스트 실행만 선택한다.
2. 최근 활동 5개와 현재 실행을 제외한다.
3. 임시 GZIP 파일을 생성한다.
4. 압축 전후 내용을 다시 읽어 SHA-256과 바이트 수가 같은지 검증한다.
5. 검증된 임시 파일을 최종 이름으로 원자 교체한다.
6. 원본 파일을 삭제한다.
7. 압축 성공·실패 이력을 서비스 로그에 누적한다.

원본과 압축 파일이 전환 순간에 함께 존재하면 조회기는 원본을 우선해 중복 집계를 막는다.

## 기존 AI 누적 로그 전환

기존 `conversations.jsonl`, `live_evaluations.jsonl`, `fault_events.jsonl`은 바로 삭제하지 않는다. 각 행의 타임스탬프를 한국 날짜로 변환해 날짜별 파일로 분리하고, 행 수와 내용 해시를 검증한 뒤 마이그레이션 manifest를 기록한다. 완료 전에는 기존 파일을 읽고, 완료 후에는 날짜별 파일만 읽어 중복을 막는다.

전환이 완료되면 기존 파일 전체는 `legacy/` 아래 GZIP 백업으로 보관한다. 날짜별 파일과 백업의 행 내용이 모두 일치하는지 확인한 뒤에만 기존 단일 파일을 제거한다.

## 보고서·대시보드·모니터링

- 라이브 보고서와 대시보드는 `.jsonl`과 `.jsonl.gz`를 함께 읽는다.
- AI 마지막 활동, 장애 retry·unavailable 복원과 VOC 프로필별 마지막 활동 복원도 압축 로그를 포함한다.
- Prometheus 시계열은 별도 영구 볼륨에 있으므로 로그 압축으로 삭제되지 않는다.
- Grafana는 Prometheus를 계속 조회하며 최신 테스트케이스 지표는 압축하지 않는 정식 보고서를 사용한다.
- Prometheus 보존 기간이 지나 삭제된 시계열을 압축 로그만으로 자동 복원하지는 않는다.

## 2026-07-18 구현·검증 결과

- 기존 AI 누적 로그를 실제 전환한 결과 대화 40행, 채점 80행, 장애 40행이 날짜별 로그에서 동일하게 조회됐다.
- 전환 후 AI 최신 보고서를 다시 생성해 CSV 80행과 요약의 대화 40건·평가 80건이 유지되는지 확인했다.
- 합성 로그로 장기간 활동이 없는 경우에도 마지막 활동 날짜 5개가 원본으로 남고 이전 날짜만 GZIP으로 전환되는지 확인했다.
- 원본과 GZIP이 함께 존재하는 전환 순간에는 원본 하나만 집계되고, 원본을 제거한 뒤에는 GZIP을 같은 데이터로 읽는지 확인했다.
- AI 배치와 VOC 프로필 실행에서 최근 완료 실행 5개는 원본으로 남고 이전 재생성용 원본만 압축되며, 보고서·그래프·manifest는 원본으로 유지되는지 확인했다.
- VOC 테스트케이스 실행 화면이 최신 Judge 원본 JSON과 압축 JSON을 모두 읽는 회귀 검사를 추가하고, 공통 JSON 조회 함수 연결을 확인했다.
- Docker 이미지 재빌드 뒤 AI API·VOC API·통합 대시보드 Health가 모두 HTTP 200이고, AI retry·unavailable과 AI·VOC 마지막 활동 지표가 압축 호환 로그에서 복원되는지 확인했다.
- 외부 AI API를 호출하지 않은 전체 회귀 결과는 `300개 통과·3개 건너뜀·2개 선택 제외`다.
