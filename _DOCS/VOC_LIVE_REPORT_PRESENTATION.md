# VOC 챗봇 최신 보고서 표시 기준

> 작성일: 2026-07-17
> 상태: **구현·누적 로그 기반 재생성·실제 보고서 화면 검증 완료**

## 1. 목적

VOC 챗봇 보고서를 AI 에이전트 챗봇 보고서와 비슷한 순서로 구성해, 긴 상세 목록을 열기 전에 전체 품질 상태와 A~D 차이를 한눈에 확인한다. 원본 대화·채점 로그는 계속 누적하고 최신 보고서만 덮어쓴다.

## 2. 보고서 순서

1. `한눈에 보는 품질 현황`
   - 전체 대화, 정상 채점, 무효 입력, 평균 처리시간
   - A~D별 대화 수, 판정 분포, N/A·미채점, 배포 가능률, 평균 총점과 처리시간
   - 9개 품질 항목의 A~D 평균 원점수
2. `품질·판정·처리시간 그래프`
   - A~D 품질 판정 분포
   - 9개 품질 항목 평균 달성률
   - A~D 평균 처리시간
3. `확인이 필요한 채점 결과`
4. `채팅 및 채점 목록`
5. `모델 프로필과 품질평가 기준`
6. `무효 입력 기록`

## 3. 긴 목록 표시 기준

- 상세 목록은 기본적으로 닫아 두고 사용자가 제목을 눌렀을 때 펼친다.
- `채팅 및 채점 목록`은 최신순 최대 50건을 표시한다.
- 목록 열은 `시각(KST)`, `모델 프로필`, `질문`, `판정`, `총점`으로 구성한다.
- `확인이 필요한 채점 결과`는 기존 질문별 결과 요약의 답변, 모델, 처리 상태, 9항목 점수·근거와 종합 근거를 그대로 포함한다.
- 정상 채점된 90점 이상 `배포 가능` 결과는 확인 목록에서 제외한다.
- 90점 미만, N/A·미채점, 처리 실패와 즉시 보류 조건은 확인 목록에 포함한다. 즉시 보류는 총점이 90점 이상이어도 제외하지 않는다.
- 손상된 과거 입력은 정상 통계와 평균에서 제외하고 별도 무효 입력 목록에 둔다.

## 4. 그래프와 저장 경로

최신 보고서와 그래프는 덮어쓰고, 이력 보고서와 그래프는 실행 시각별로 보존한다.

```text
_OUTPUT/reports/voc/live/latest/voc_live_report.md
_OUTPUT/reports/voc/live/latest/assets/*.png
_OUTPUT/reports/voc/live/history/voc_live_report_{실행시각}.md
_OUTPUT/reports/voc/live/history/assets/{실행시각}/*.png
```

그래프는 누적 JSONL 로그만 읽어 생성하며 외부 AI API를 호출하지 않는다. N/A·미채점은 0점으로 환산하지 않고 품질 평균에서 제외한다. 품질 달성률 축은 0~100%, 판정 분포 축은 정수 건수로 표시한다.

## 5. 구현·검증 결과

- 기존 누적 원본 로그 8건을 사용해 최신 보고서와 PNG 3개를 재생성했다.
- 현재 로그 기준 유효 대화는 7건, 정상 채점은 6건, 무효 입력은 1건이며 확인 필요 목록은 7건이다.
- 최신·이력 Markdown과 각각의 이미지 자산 경로를 분리했다.
- 보고서 생성 테스트에서 95점 정상 결과가 확인 필요 상세에서 제외되고 채팅 목록에는 유지되는 것을 검증했다.
- 실제 외부 AI API는 호출하지 않았다.
