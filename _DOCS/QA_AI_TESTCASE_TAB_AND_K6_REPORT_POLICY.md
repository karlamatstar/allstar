# QA AI 테스트케이스 탭·K6 보고서 정책

> 확정일: 2026-07-17
> 상태: **구현·비API 자동 검증 완료**
> 적용 대상: `tools/qa_control/`, QA 공통 로그·보고서 자동화

## 1. 목적

QA 컨트롤러의 AI 상담 품질검사 마지막에 AI 테스트케이스 정식 시험을 추가한다. 동시에 K6를 사용한다는 이유만으로 기존 장애·성능 정식 보고서를 제거하지 않도록 시험별 결과 보존 범위를 명확히 구분한다.

2026-07-17 구현에서 AI 테스트케이스 품질 시험 탭을 추가했고, 직접 K6 5종은 사용자용 공통 Markdown·manifest 생성 대상에서 제외했다. 기존 장애·기능 검증 및 서버 연결 성능 종합 시험의 보고서 흐름은 유지했다.

## 2. 통합 대시보드 범위

- 통합 대시보드에 별도의 `QA 실행 이력`·필터 화면을 추가하지 않는다.
- 현재 대시보드의 챗봇·테스트케이스·보고서·Grafana 화면을 유지한다.
- QA 실행 증적은 사용자용 신규 화면을 만들지 않더라도 문제 추적을 위해 원본 로그에 계속 누적한다.
- `run_id`와 공통 실행 이벤트는 내부 증적 연결에 사용하며 일반 대시보드 표에는 노출하지 않는다.

## 3. AI 테스트케이스 품질 시험 탭

AI 상담 품질검사의 가장 마지막에 다음 두 줄 탭을 추가한다.

```text
테스트케이스 품질 시험
(Test Case Test)
```

화면은 기존 시험 탭과 같은 구성으로 유지한다.

- 시험 설명
- 현재 등록된 AI 테스트케이스 전체 수
- 실제 외부 API 호출과 비용 발생 가능성 안내
- `실행` 버튼
- `중지` 버튼
- 아래쪽 터미널 출력 화면
- 전체 QA 탭 공통 단일 실행 잠금

사용자가 QA 컨트롤러에서 실행하면 등록된 AI 테스트케이스 전체를 사용한다. 실행 전 확인창에서 전체 사례 수와 예상 외부 AI 호출 범위를 안내한다. 명령행 검증도 실행 목적에 맞는 사례 범위를 명시하고 2건으로 고정하지 않는다.

정상 완료하면 기존 AI 테스트케이스 파이프라인이 다음 정식 결과를 갱신한다.

- 규칙 기반·API 기반 비교 JSON
- 비교 CSV
- `AI 챗봇 품질관리 최종 비교 보고서` Markdown
- 테스트케이스별 총점, 품질 항목 평균, 판정 분포 PNG 그래프
- 실행 시각별 이력본

사용자 중지·실행 실패·시작 실패 시에는 터미널 출력과 원본 실행 로그만 새 실행 증적으로 남긴다. 마지막 정상 완료 정식 보고서와 최신 그래프는 실패 실행으로 덮어쓰지 않는다.

## 4. K6 시험별 보고서 보존 정책

### 4.1 Grafana 중심 직접 부하 시험 5개

다음 시험은 별도의 사용자용 Markdown·Word·PNG 보고서를 만들지 않는다.

1. 기본 동작 시험 (Smoke Test)
2. 일반 부하 시험 (Load Test)
3. 무작위 요청 시험 (Random Test)
4. 한계 부하 시험 (Stress Test)
5. 순간 급증 시험 (Spike Test)

다음 실행 증적과 모니터링 데이터는 유지한다.

- QA 컨트롤러 터미널 출력
- 실행별 누적 원문 로그
- 필요 시 K6 원본 요약 JSON
- Prometheus remote write
- 실행별 `testid`
- Grafana `K6 성능 부하 시험` 패널

### 4.2 기존 정식 보고서를 반드시 유지하는 K6 복합 시험

다음 시험은 K6를 사용하지만 단순 부하 그래프가 목적이 아니므로 기존 정식 보고서를 제거하지 않는다.

| 시험 | 유지할 결과 |
|---|---|
| 장애·기능 검증 시험 | 장애 모의 K6와 비AI Pytest 결과, 결함 Markdown·Word 보고서 |
| 서버 연결 성능 종합 시험 | 1명·10명·25명 단계별 독립 K6 원본과 서버 연결 성능 Markdown 보고서 |

2026-07-17 후속 정리에서 인위적 503 장애를 만들던 서버 연결 끊김 방어 시험은 제거했다. VOC A~D 정식 보고서와 AI·VOC 챗봇 보고서의 기존 기록 흐름은 유지한다.

## 5. 구현·검증 기준

- AI 상담 품질검사 8개 탭 중 테스트케이스 품질 시험이 실제 마지막으로 표시된다.
- 기본 창과 최소 해상도에서 신규 탭이 잘리지 않는지 확인한다.
- 신규 탭도 실행 중 다른 모든 시험의 실행 버튼을 잠그고 자기 중지 버튼만 활성화한다.
- 확인창을 승인하기 전에는 외부 AI 프로세스를 시작하지 않는다.
- 직접 부하 K6 5개는 정식 사용자 보고서를 생성하지 않지만 로그와 Grafana 데이터는 남겨야 한다.
- 장애·기능 검증 및 서버 연결 성능 보고서는 기존 파일명·대시보드 표시와 함께 유지해야 한다.
- 실패·중지된 AI 테스트케이스 실행이 마지막 정상 정식 보고서를 변경하지 않는지 자동 테스트로 확인한다.
- 구현 검증에서는 외부 AI API를 호출하지 않았다.

## 6. 구현 검증 결과

- QA 컨트롤러·공통 보고서·정식 보고서 보호 집중 테스트 `30개 통과`
- 실제 외부 AI 호출 파일과 서버 종단 파일을 제외한 전체 회귀 `239개 통과, 2개 환경 제외`
- 직접 K6 5종의 원문 로그·K6 JSON·완료 이벤트 지표 유지와 Markdown·manifest 미생성을 자동 검증
- 장애·기능 검증과 서버 연결 성능 종합 시험의 공통 실행 요약 생성 유지 검증
- AI 테스트케이스 실행 실패 시 기존 `final_quality_report.md`가 변경되지 않는지 자동 검증
- Docker·서버를 시작하지 않았고 외부 AI API를 호출하지 않음
