# AI 에이전트 챗봇·품질 화면 후속 개선

## 1. 문서 상태

- 작성일: 2026-07-17
- 대상: 통합 대시보드의 `AI 에이전트 챗봇`, `AI 에이전트 테스트케이스`
- 현재 상태: 구현·집중 비API 시험·브라우저 화면 검증 완료
- 실제 외부 AI API 검증: UI 검증에는 사용하지 않음. 전체 시험 명령에 실제 API 시험이 포함되어 별도 호출된 사실은 7절에 기록

## 2. AI 에이전트 챗봇 배치

`챗봇과 대화` 화면은 다음 순서를 사용한다.

1. 필수 체크 사항
2. 채팅 내역
3. 메시지 입력창

필수 체크 사항은 채팅창 바로 위에 두고 채팅창과 입력창 사이에 다른 안내 영역을 넣지 않는다. 사용자가 입력한 메시지는 오른쪽, AI 에이전트 답변은 왼쪽에 배치한다.

질문을 전송하면 사용자 메시지를 채팅창에 먼저 표시한다. 서버 응답을 기다리는 동안 채팅창의 왼쪽 답변 위치에 `답변을 입력하고 있습니다` 상태를 표시하고, 답변이 도착하면 해당 상태를 제거한 뒤 실제 답변으로 교체한다.

## 3. 품질 현황 비교 그래프

### 3.1 공통 원칙

- 동일 질문 또는 동일 테스트케이스의 서버 연결 방식(API)과 규칙 기반 결과를 위아래로 누적하지 않는다.
- 한 질문·케이스를 한 묶음으로 만들고 두 결과를 좌우 막대로 표시한다.
- 막대 높이는 종합점수, 막대 색상은 `PASS`, `REVIEW`, `FAIL`, `N/A` 판정을 뜻한다.
- 마우스를 올리면 비교 대상, 답변 종류, 판정, 종합점수, 평가 요약을 확인할 수 있게 한다.
- 대화·케이스 수와 평가 결과 행 수를 구분해 표시한다.

### 3.2 표시 범위와 이동

- 기본 표시 개수: 최근 질문 또는 현재 보고서의 테스트케이스 10개
- 선택 항목: 5개, 10개, 20개, 전체
- `이전`, `다음` 버튼으로 페이지를 이동한다.
- 질문 한 건과 테스트케이스 한 건에 속한 두 평가 결과는 페이지가 달라져도 분리하지 않는다.

실시간 챗봇은 최근 질문부터 표시한다. 배치 보고서는 현재 보고서의 테스트케이스 순서를 유지한다.

## 4. 유형별 비교

AI 에이전트 챗봇과 AI 에이전트 테스트케이스의 `유형별 비교`에 같은 구성 원칙을 적용한다.

- PC: 왼쪽 레이더 그래프, 오른쪽 정확한 평균점수 표
- 좁은 화면: 레이더 그래프 다음에 점수표를 위아래로 배치
- 표 행: 정확성, 근거성, 유용성, 안전성, 이해 가능성, 품질 평균
- 표 열: 서버 연결 방식(API), 규칙 기반, 두 결과의 점수 차이
- 점수 범위: 0~5점
- `N/A`와 미채점 결과는 평균에서 제외
- 모델별 평균에 사용한 평가 건수 표시
- 각 행에서 더 높은 모델 점수를 강조

품질 항목의 의미는 다음과 같다.

| 품질 항목 | 의미 |
|---|---|
| 정확성 | 질문에 사실상 정확하게 답했는지 |
| 근거성 | 제공된 자료와 근거에 기반한 답변인지 |
| 유용성 | 사용자의 문제 해결에 도움이 되는지 |
| 안전성 | 위험하거나 부적절한 내용이 없는지 |
| 이해 가능성 | 표현이 명확하고 읽기 쉬운지 |

## 5. 상세 화면

`대화별 채점 상세`와 `케이스 상세`는 같은 화면 형식을 유지한다.

- 공통: 판정 필터, 상세 표, 종합점수, 품질 항목별 점수, 판정, 평가 요약
- 대화별 채점 상세: 질문, 실제 답변, 응답시간, 답변 종류 중심
- 케이스 상세: 테스트케이스 ID, 분류, 테스트 유형, 실행 결과 중심
- `request_id`는 사용자 화면에서 숨기고 원본 로그 연결 용도로만 유지

## 6. 검증 기준

- 실제 외부 AI API를 호출하지 않는 구조·화면 시험을 우선한다.
- 저장된 실시간 보고서 표본으로 질문별 좌우 막대와 페이지 이동을 확인한다.
- 저장된 데이터 또는 시험용 표본으로 배치 품질 현황의 케이스별 좌우 막대를 확인한다.
- 두 유형별 비교 화면에서 레이더와 정확한 점수표가 함께 표시되는지 확인한다.
- 채팅 응답은 가짜 지연 응답을 사용해 사용자 메시지 선표시와 입력 중 상태 교체를 검증한다.

## 7. 구현·검증 결과

- AI 챗봇의 필수 확인 상자를 채팅 영역 위로 이동하고 520px 채팅창과 입력창의 간격을 0px로 연결했다.
- 사용자 메시지 우측 정렬, AI 답변 좌측 정렬 CSS와 채팅창 내부 입력 중 상태를 구현했다.
- AI 답변 요청을 백그라운드 작업으로 분리해 사용자 메시지를 먼저 화면에 반영한다.
- 실시간 품질 현황에서 대화 수와 평가 결과 수를 분리하고 최근 질문 10개를 기본값으로 적용했다.
- 저장된 대화 12건·평가 24건 표본에서 `1 / 2` 페이지와 이전·다음 이동을 확인했다.
- 질문 하나당 두 막대, 총 10개 질문의 20개 막대가 좌우로 분리된 좌표를 브라우저에서 확인했다.
- 유형별 비교는 PC에서 레이더와 평균점수 표가 좌우로 표시되며, 평가 건수·점수 차이·항목 설명을 함께 표시한다.
- 배치 품질 현황과 배치 유형별 비교도 실시간 화면과 같은 공통 함수를 사용한다.
- 집중 비API 구조·동작 시험 29건이 통과했다.
- 실제 API 시험 파일과 실행 중인 VOC 서버 종단 시험을 제외한 회귀 시험은 `193 passed`로 통과했다.
- 최초 전체 시험 명령에는 실제 OpenAI 시험인 `test_evaluation_pipeline.py` 대표 2건과 `test_negative_cases.py` 1건이 포함됐다. TC-026은 안전한 거절 답변을 반환했지만 기대 키워드 정규식에 없는 표현을 사용해 실패했다.
- 당시 VOC 6개 에이전트 서버도 실행 중이어서 VOC MCP·종단 시험이 건너뛰어지지 않았다. 이 혼합 시험 결과는 `214 passed, 1 skipped, 1 failed`이며 UI 합격 기준에는 사용하지 않았다.
- 이후 검증은 위 실제 API 시험 2개 파일과 VOC MCP·종단·장애 시험 3개 파일을 명시적으로 제외했다.
