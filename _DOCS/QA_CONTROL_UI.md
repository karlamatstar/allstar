# 품질검사 관리 GUI 화면 기준

> 최초 작성일: 2026-07-17

## 탭 표시 원칙

- 탭은 쉬운 한국어 기능명을 첫째 줄에 표시한다.
- 영문 전문용어가 필요하면 괄호와 함께 둘째 줄에 표시한다.
- 선택 탭과 비선택 탭은 `14×9`의 동일한 내부 여백을 사용한다.
- 선택 상태는 탭 크기를 바꾸지 않고 배경색과 글자색으로만 구분한다.
- 탭의 글자는 가로·세로 중앙에 정렬한다.

최상위 탭은 다음처럼 표시한다.

```text
AI 상담 품질검사
(AI Agent QA)

고객 의견 분석 품질검사
(VOC QA)
```

AI 상담 품질검사의 영문 시험명도 `기본 동작 시험`과 `(Smoke Test)`처럼 두 줄로 나눈다. 장애 모의 k6, 전체 기능 검사 pytest, 결함 보고서 생성을 함께 수행하는 탭은 `장애·기능 검증 시험`과 `(Validation Test)`로 표시한다.

## 고객 의견 분석 A~D 탭

A~D는 단순히 `A 테스트`로 표시하지 않고 다음 형식을 사용한다.

```text
에이전트 교차 테스트
(A)
```

B, C, D도 같은 형식으로 표시한다. 탭 안쪽의 화면 제목은 `에이전트 교차 테스트 (A)`처럼 한 줄로 표시하고, 답변 생성 모델·독립 품질 평가 모델·추론 강도 설명은 기존대로 유지한다.

## 실행 버튼과 결과 위치

- 각 탭에는 `실행`, `중지` 버튼만 둔다.
- `보고서 폴더` 버튼은 제공하지 않는다.
- 실행 결과는 아래 실행 기록 화면에 표시한다.
- 생성된 보고서는 `_OUTPUT/reports/` 구조에 저장한다.

GUI 화면에서는 보고서의 파일 경로나 폴더 위치를 표시하지 않는다. 이후 대시보드에서 보고서를 확인하는 흐름을 기준으로 하며, 각 탭의 상단 설명 영역은 저장 위치 대신 다음 내용을 안내한다.

- 이 시험이 어떤 방식으로 요청을 발생시키는지
- 이 시험으로 어떤 상태와 품질을 확인할 수 있는지
- 시험 중 어떤 단계가 진행되는지
- 결과 또는 보고서가 자동으로 정리·생성되는지

기본 동작·일반 부하·무작위 요청·한계 부하·순간 급증·장애 및 기능 검증·서버 연결 성능·서버 연결 끊김 방어 시험은 각각 별도의 설명을 사용한다. VOC 전체 비AI 검사·단위 테스트·A~D 에이전트 교차 테스트에도 해당 실행 범위에 맞는 설명을 표시한다.

## 부하 시험 입력값

이전 `ai_agent_quality_portfolio/RUN/test_launcher.py`의 부하 설정을 유지한다.

| 시험 | 기본 최대 가상 인원(VU) | 기본 실행 시간 |
|---|---:|---:|
| 일반 부하 시험(Load Test) | 20명 | 60초 |
| 무작위 요청 시험(Random Test) | 100명 | 60초 |
| 한계 부하 시험(Stress Test) | 100명 | 120초 |
| 순간 급증 시험(Spike Test) | 200명 | 60초 |

- 네 시험 탭에는 `최대 가상 인원(VU)`과 `실행 시간(초)` 입력란을 표시한다.
- 두 값은 모두 1 이상의 정수만 허용한다.
- 실행 전 확인 창과 실행 기록에 실제 적용값을 표시한다.
- 실행 프로세스에는 `K6_VUS`, `SCRIPT_DURATION`, `TARGET_IP=127.0.0.1:8000` 환경변수로 전달한다.
- 기본 동작 시험(Smoke Test)은 VU 1 고정이므로 입력란을 표시하지 않는다.

## 2026-07-17 검증

- Tkinter GUI를 숨김 상태로 생성해 최상위·AI 시험·VOC 시험 탭의 실제 두 줄 문자열을 확인했다.
- 선택·비선택 탭의 실제 여백 값이 모두 `(14, 9)`인지 확인했다.
- QA 화면 전용 테스트를 포함한 통합 테스트 결과는 구현 기준서의 최신 검증 기록을 따른다.
- 실제 외부 AI API는 호출하지 않았다.
