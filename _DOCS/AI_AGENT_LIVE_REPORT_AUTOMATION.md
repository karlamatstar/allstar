# AI Agent 실시간 채팅 로그·보고서 자동화 기준

> 최초 작성일: 2026-07-17  
> 적용 대상: `_Total` AI 상담 에이전트의 `/chat` 실시간 대화

## 목적

사용자가 AI Agent 챗봇과 대화할 때 답변과 채점 로그를 누적하고, 별도의 수동 생성 요청 없이 채점 완료 직후 최신 실시간 품질 보고서를 자동 갱신한다. 보고서 때문에 사용자 답변이 늦어지지 않도록 답변 반환과 채점·보고서 생성을 분리한다.

## 처리 순서

1. 사용자가 `/chat`으로 질문한다.
2. API 기반 답변과 비교용 규칙 기반 답변을 먼저 반환한다.
3. 질문·답변·응답시간·`request_id`를 대화 JSONL에 누적한다.
4. 백그라운드에서 API 기반 답변과 규칙 기반 답변을 각각 독립 채점한다.
5. 두 채점 결과를 `request_id`로 연결해 채점 JSONL에 누적한다.
6. 두 채점 로그가 저장된 직후 최신 Markdown·CSV·PNG 보고서를 자동 갱신한다.

사용자에게 답변이 표시된 뒤 채점이 끝나야 보고서가 바뀌므로 화면에는 몇 초의 차이가 생길 수 있다. 보고서 생성 실패는 채팅 답변과 누적 원본 로그를 취소하지 않는다.

## 저장 구조

```text
_OUTPUT/logs/ai_agent/live/conversations/conversations.jsonl
_OUTPUT/logs/ai_agent/live/judgments/live_evaluations.jsonl

_OUTPUT/reports/ai_agent/live/live_report.md
_OUTPUT/reports/ai_agent/live/live_report.csv
_OUTPUT/reports/ai_agent/live/assets/decision_distribution.png
_OUTPUT/reports/ai_agent/live/assets/quality_axis_average.png
_OUTPUT/reports/ai_agent/live/assets/response_latency_trend.png
```

- 대화와 채점 JSONL은 삭제하거나 덮어쓰지 않고 계속 누적한다.
- Markdown·CSV·PNG는 누적 로그 전체를 다시 집계한 최신본으로 교체한다.
- 매 채팅마다 거의 같은 보고서 이력 사본을 만들지 않는다. 원본 JSONL이 시간별 증적 역할을 한다.
- 보고서 파일과 PNG는 임시 파일을 완성한 뒤 교체하여 작성 중인 불완전한 파일 노출을 줄인다.
- 한 프로세스 안에서 여러 채팅이 동시에 완료되어도 보고서 생성은 한 번에 하나씩 진행한다.

## 보고서 구성

보고서를 열었을 때 긴 대화 목록보다 전체 상태를 먼저 판단할 수 있도록 다음 순서를 사용한다.

1. 한눈에 보는 품질 현황
   - 집계 기간, 대화 수, 평균·최대 응답시간
   - 모델별 PASS·REVIEW·FAIL·N/A·미채점 건수
   - 통과율, 평균 종합점수, 품질 항목 평균점수
2. 품질·판정·응답시간 그래프
   - 모델별 채점 판정 분포
   - 정확성·근거성·유용성·안전성·이해가능성 평균점수
   - 최근 대화 최대 30건의 응답시간 추이
3. 확인이 필요한 채점 결과
   - FAIL·REVIEW·N/A 사례만 접기·펼치기로 제공
4. 채팅 및 채점 목록
   - 최근 50개 평가 행을 접기·펼치기로 제공

그래프는 AI 이미지 생성이 아니라 누적 로그 수치를 Pillow로 그린 PNG다. N/A와 미채점은 FAIL로 합치지 않고 별도 항목으로 표시한다. 품질 평균과 통과율은 실제로 채점 가능한 PASS·REVIEW·FAIL만 집계한다.

## 대시보드 동작

- 통합 대시보드는 최신 Markdown과 PNG를 함께 표시한다.
- 새 대화가 보고서보다 최신이면 백그라운드 채점·자동 갱신 중일 수 있다는 안내를 표시한다.
- `실시간 보고서 다시 갱신` 버튼은 자동화 실패나 즉시 재집계가 필요할 때만 사용하는 보조 기능이다.
- 대시보드 파일 캐시는 최대 10초 뒤 최신 결과를 다시 읽으며, 사용자는 `새로고침`으로 즉시 다시 읽을 수 있다.

## 오류 처리

- API 답변 생성 실패 또는 Judge 호출 실패는 품질 FAIL이 아니라 N/A로 기록한다.
- 예상하지 못한 채점 예외도 N/A 채점 로그로 남겨 대화와 채점 행이 끊기지 않게 한다.
- 보고서 갱신 오류는 서버 로그에 남기고 다음 채팅 완료 시 다시 자동 갱신한다.
- JSONL의 마지막 행을 다른 요청이 쓰는 순간 읽어 파싱할 수 없으면 해당 행은 다음 자동 갱신에서 반영한다.

## 2026-07-17 검증

- 실제 AI API를 호출하지 않는 가짜 대화 2건·평가 행 4건으로 검증했다.
- Markdown·CSV 최신본 생성, PNG 3개 생성, 접이식 상세 목록을 확인했다.
- 두 모델의 채점 로그 뒤 보고서 갱신이 호출되는 순서를 자동 테스트로 확인했다.
- N/A와 미채점 분리, 통과율·평균점수 분모 제외를 자동 테스트로 확인했다.
- AI Agent 전용 비API 테스트 10개가 통과했다.
- 전체 비API 회귀는 144개 통과, 환경 조건 3개 건너뜀, 실제 API 종단 테스트 2개 선택 제외로 완료했다.
- 실제 OpenAI·Anthropic 답변·채점 API는 호출하지 않았다.
